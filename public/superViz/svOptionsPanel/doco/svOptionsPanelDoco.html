svOptionsPanel Directive

A control-panel may have a closed state and an open state. These two states can be expressed in a single boolean variable exposed in the scope, which we will call "show".

Usage

  • Code:
    <!--Within the scope of the superVis directive include the following:-->

        <sv-options-panel-trigger></sv-options-panel-trigger>
        <sv-options-panel></sv-options-panel>
  • svOptionsPanelTrigger Output

  • svOpotionsPanel Output (This is contracted by default. Click the trigger button above to expand.)

  • propertyBag: (This is bound to what is actually on this page. Notice the value of panelExpanded when triggered above.)


    {{propertyBag}}

Rules

  • The svOptionsPanel directive is a child of the superViz directive. It is responsible for displaying options that are specific to the datatype assigned to it's parent superViz.

  • The svOptionsPanel is the parent of the visuzlizationtype-specific directive, sv-options-[visualizationType].

  • The svOptionsPanel shares scope with it's parent superViz.

  • The datatype associated with a svOptionsPanel (or a superViz, for that matter) never changes. If a user wants a different datatype associated with a chart, s/he must delete the chart and drag a different question onto the grid.

  • The visualization type associated with a svOptionsPanel can change, since a user may choose to change a vizualization between different types (say, from verticalbar to linechart).

  • The svOptionsPanel is the parent of svOptions[visualization type]. This means that depending on the datatype associated with the current chartContainer, a corresponding svOptionsPanel[visualization type] directive is instanciated. For example, if the current datatype is SingleChoice, and the default visualization type for SingleChoice is PieChart, the svOptionsPanel will instanciate a child sv-options-piechart directive.

  • svOptionsPanel Sizing -- Most of the time the svOptionsPanel is hidden. When it is activated it should float along the upper inside right edge of the parent superViz directive.

superVis Scope-level Properties - The scope.propertyBag object

The shared properties within the superVis directive (and all it's children) should be appended to the propertyBag object. This is originally added to the superViz directive.

  • panelExpanded

    TRUE | FALSE -- If true, expand (show) the panel. Otherwise, contract (hide) it. Trigger animation to expand | contract

  • visualizationType

    String corresponding to valid visualization type defined in MetricMakerSrvc.visualizationTypes 

    MetricMakerSrvc.visualizationTypes={{MetricMakerSrvc.visualizationTypes}}

Diagrams

  • svOptionsPanelTrigger directive -- provides the mechanism to trigger the expansion of the OptionsPanel. This is a seperate directive so it can be placed in a SuperViz header while the panel is placed somewhere else. As long as they share the same scope it will work.
